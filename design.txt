README DESIGN BLUEPRINT (Abunesh126)
=================================

Goal
----
Build a premium GitHub profile README with strong visual identity, smooth animated elements, and clean section flow inspired by the provided designs.

Core Design Style
-----------------
1) Theme
   - Dark background visual tone (#0d1117 style)
   - Neon accent highlights (purple / cyan / pink)
   - High contrast text for readability

2) Typography Feel
   - Centered hero heading
   - Friendly intro line + role line (AI/Data/Full-Stack)
   - Short, punchy bullets in sections

3) Layout Rhythm
   - Hero banner
   - Intro animation
   - CTA badges (Portfolio / LinkedIn / Email / Profile views)
   - About section
   - Learning + Focus sections
   - Skills and tools icons grid
   - Projects / Internship / Goals
   - GitHub stats + streak + activity graph
   - Footer quote or signature

Animation & Dynamic Elements
----------------------------
1) Typing animation (readme-typing-svg)
   - Primary heading typing line
   - Secondary motivation/mission typing line

2) Coding GIF block
   - Use animated coding image beneath heading
   - Keep centered and width-balanced (around 380-420)

3) Optional dynamic extras
   - Contribution snake animation (if desired)
   - Pacman contribution graph (optional alternative)

Visual Components to Keep
-------------------------
1) Header banner image
   - Local custom banner or generated capsule banner

2) Badge row (primary CTA)
   - Portfolio button
   - LinkedIn button
   - Email button
   - Profile views counter

3) Skills visualization
   - Skillicons for language/tool logos
   - Optional shields for specific technologies

4) Stats visualization
   - GitHub readme stats card
   - Top languages card
   - Streak stats card
   - Activity graph card

Content Design Structure
------------------------
1) About Me
   - 3 to 5 bullets only
   - Focus on identity, strengths, and mission

2) Currently Learning
   - Keep future-facing learning areas
   - Use practical engineering language

3) Skills
   - Group by category:
     Programming / Frontend / Backend / Database / AI & Data / Tools

4) Project Highlights
   - 5 to 7 projects max
   - One-line impact statement each

5) Internship / Experience
   - Role + organization + 2 impact bullets

6) Collaboration / Opportunities
   - Open-to-collab statement
   - Areas of contribution

7) Career Objective + Future Goals
   - Keep concise and actionable

Recommended Color Direction
---------------------------
- Background: deep dark
- Accent 1: purple (innovation, AI tone)
- Accent 2: cyan or blue (tech/data feel)
- Accent 3: pink/red (highlight contrast)
- Keep consistent accents across badges and stat themes

Markdown Implementation Rules
-----------------------------
1) Keep major section headers with emojis for scanability
2) Use horizontal separators between major blocks
3) Prefer centered HTML blocks only for visual hero/stats elements
4) Keep body sections in clean markdown bullets
5) Avoid overloading too many moving widgets in one area

Performance & Readability Balance
---------------------------------
1) Prioritize readability over too many visual embeds
2) Use only stable, commonly used badge/stat URLs
3) Ensure links are valid and updated
4) Keep file clean and maintainable

Final Build Checklist
---------------------
[ ] Header banner configured
[ ] Typing hero line working
[ ] Coding GIF placed
[ ] Portfolio/LinkedIn/Email/Profile badges working
[ ] About + Learning + Skills completed
[ ] Projects and experience summarized
[ ] GitHub stats/streak/activity graph embedded
[ ] Quote/footer added
[ ] All links tested
[ ] Visual consistency verified

Design Outcome
--------------
A polished, modern, animation-rich profile README that feels professional, energetic, and AI/data-engineering focused while staying readable and recruiter-friendly.
